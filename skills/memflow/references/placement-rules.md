# memflow placement rules

## Daily note
Use for:
- raw session events
- temporary findings
- trial and error
- work logs
- unfinished hypotheses

## MEMORY.md
Use for:
- stable user preferences
- recurring operational patterns
- long-lived architecture decisions
- cross-session facts worth preserving

## TOOLS.md / topical docs
Use for:
- environment-specific lookup facts
- host aliases
- domains, ports, services
- voice mappings
- operational entry points

## Archive
Use for:
- superseded notes
- old experiments
- historical details no longer needed in primary memory

## Conflict rules
- newer verified fact beats older note
- compress process into conclusion
- do not leave contradictions in long-term memory
